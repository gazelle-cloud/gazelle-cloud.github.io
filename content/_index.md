# What is Gazelle?

This is a personal playground for diving into the Azure Landing Zone concept — with everything fully automated. The goal? Build something that works better together than it would in pieces. All the code is on GitHub, open for anyone to explore, use, or build on. The platform is shaped by a few core principles:

- **Cost efficiency**: Pay for what you use—nothing more, nothing less. If a platform is handling thousands of operations, it should reflect in the cost. But when it's sitting idle, it should cost nothing. Simple as that. 

- **Frictionless flow**:  Engineers should be able to spin up and tear down their own landing zones without any bottlenecks. Just a smooth, frictionless self-service experience. Fewer shared services, more freedom to get things done.

- **Embrace Change**: The platform should make change easy to apply. A quick tweak shouldn't turn into a project, and even major updates should remain manageable. Change should be a routine part of working with the system, not an exception.
